# 橘猫和贵宾 - AI短剧制作工作流

## 项目概述

用 AI 生成一对角色（橘猫大哥 + 贵宾小姐）的系列短视频，实现：
- 固定角色形象
- 克隆配音
- 图生视频推进剧情

## 完整工作流

```
┌─────────────────────────────────────────────────────────────┐
│                    AI短剧制作流程                            │
└─────────────────────────────────────────────────────────────┘

Step 1: 剧本创作
├── 写故事大纲
├── 分镜头剧本（每个镜头5秒）
├── 角色台词
└── 输出：剧本/第X集_标题.md

        ↓

Step 2: 语音合成（TTS）
├── 提取台词文本
├── 调用 MiniMax T2A 接口
├── 使用克隆的 voice_id
└── 输出：音频/第X集/角色_序号.mp3

        ↓

Step 3: 视频生成（图生视频）
├── 准备起始帧图片
├── 编写动作描述 prompt
├── 调用 MiniMax Video API
├── 每段约5秒
└── 输出：视频/第X集/V序号.mp4

        ↓

Step 4: 音视频合成
├── 将语音对齐到视频时间轴
├── 添加字幕
├── 添加背景音乐
├── 调整音量平衡
└── 输出：成品/第X集_完整版.mp4

        ↓

Step 5: 后期调整
├── 检查口型同步
├── 调整节奏
├── 添加转场
└── 输出：成品/第X集_终版.mp4
```

## API 调用参考

### 语音合成（TTS）

```bash
curl -X POST "https://api.minimax.chat/v1/t2a_v2?GroupId=xxx" \
  -H "Authorization: Bearer API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "speech-02-hd",
    "text": "台词内容",
    "voice_setting": {
      "voice_id": "shehui_dage_jumu_002"
    }
  }'
```

### 视频生成（图生视频）

```bash
curl -X POST "https://api.minimax.chat/v1/video_generation?GroupId=xxx" \
  -H "Authorization: Bearer API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "video-01",
    "prompt": "动作描述",
    "first_frame_image": "base64编码的图片"
  }'
```

## 注意事项

### ⚠️ 核心原则：角色一致性控制（重要！）

通过实践摸索出的关键经验：

1. **人物不能离开镜头**
   - 两个角色必须始终保持在画面中
   - 一旦某个角色离开镜头，下一次生成时AI会按自己的想象重新设计，导致角色外观失控

2. **固定机位原则**
   - 这类双人对话视频必须采用固定机位
   - 即使是行走场景，也要让背景跟着人物移动，而不是让人物走出画面

3. **切换回原图的时机**
   - 如果画面中两个人物变成一个人物，必须切换回原始起始帧重新生成
   - 否则角色一致性会彻底崩坏

4. **Prompt 写法建议**
   - 明确描述"两个角色始终在画面中"
   - 使用"camera follows"而不是"character walks away"
   - 避免任何让角色离开镜头的动作描述

### 其他注意事项

1. **口型同步**：AI视频的嘴部动作可能与语音不完全匹配
2. **临时音色**：克隆的音色48小时内需使用一次才能永久保留
3. **视频时长**：MiniMax 单次生成约10秒，需要分段拼接
